مرحبًا،

أرسل لك هنا وثيقة متطلبات تفصيلية (بصيغة رسالة واحدة) لتطوير منصة الأكاديمية الخاصة بنا. المطلوب تنفيذ منصة “مساعدات المعلمين الأونلاين” تساعد المعلم على التطوير من خلال كورسات مسجّلة + منهج تفاعلي، وتساعد أصحاب الأكاديميات في تقييم وتدريب وتوظيف المعلمين، مع نظام دعم وتقارير ومدفوعات وشهادات متعددة الأكاديميات.

=====================================================

1) فكرة المنصة باختصار (Product Idea)
=====================================================
المنصة هدفها:

- تطوير وتأهيل المعلمين الأونلاين عبر:
  1) كورسات مسجلة + اختبارات بعد كل فيديو
  2) منهج تفاعلي (مقسّم لمستويات وإرشادات)
  3) تقارير أسبوعية لمتابعة التقدّم
  4) خدمة شهادات لكل أكاديمية بقوالب مختلفة
  5) نظام اجتماعات مباشر (بديل Zoom) + سبورة + تقرير + ملخص للحصة
  6) دليل معلمين (بروفايلات + فيديو تعريفي) يمكن لأصحاب الأكاديميات الوصول له والتواصل مع المعلمين من خلاله مع فلاتر بحث قوية
  7) نظام مدفوعات وتسجيل تلقائي لكل عملية دفع داخل الموقع
- مساعدة أصحاب الأكاديميات في:
  - الوصول لمعلم مناسب بسرعة عبر الفلاتر
  - التواصل مع المعلم داخل المنصة
  - متابعة تقارير ومؤشرات أداء المعلمين التابعين لهم (إن كانوا ضمن أكاديميتهم)

المنصة ستكون Multi-Academy (Multi-tenant):

- يوجد عدة “أكاديميات” داخل نفس النظام
- بيانات كل أكاديمية تكون منفصلة: شهادات، معلمين تابعين لها، تقاريرها… إلخ
- المدير (Admin/Owner) يستطيع رؤية وإدارة الكل.

=====================================================
2) الأدوار وصلاحيات كل دور (Roles & Permissions)

المطلوب نظام صلاحيات RBAC + Service Entitlements (خدمات مفعلة لكل معلم)، بحيث لا يظهر للمعلم إلا الخدمات التي تم تفعيلها له.

الأدوار الأساسية:

1) المدير (Admin / Owner):

- تحكم كامل في كل شيء داخل النظام
- إنشاء وإدارة الأكاديميات (Tenants)
- إدارة المستخدمين والأدوار
- إعداد الخدمات/الباقات وتفعيلها للمستخدمين
- إدارة المناهج والكورسات المسجّلة
- إدارة قوالب الشهادات لكل أكاديمية
- رؤية التقارير والتحليلات كاملة
- إدارة المدفوعات وسجلاتها
- إدارة لوحة الدعم/الأعطال/التذاكر (إن تم تنفيذها في المرحلة الأولى أو الثانية)

2) المحاسب (Accountant):

- صلاحيات مالية فقط:
  - الاطلاع على كل عمليات الدفع (Transactions) المسجلة تلقائيًا
  - تقارير الإيرادات، الاستردادات (Refunds) إن وُجدت
  - تصدير تقارير Excel/PDF
- بدون صلاحية تعديل المحتوى أو المستخدمين.

3) المشرف العام (Supervisor):

- يتحكم في حسابات المعلمين:
  - إنشاء حسابات/تفعيل/تعطيل
  - تخصيص الخدمات لكل معلم (لو عندنا 3 خدمات والمعلم مشترك في 2 → يظهر له فقط 2)
- الاطلاع على تقارير تقدم المعلمين.

4) المعلم (Teacher):

- المستفيد من الخدمات:
  - منهج تفاعلي (حسب المستوى)
  - كورسات مسجلة + اختبارات + تقييم
  - اجتماعات مباشرة + سبورة + تقارير
  - شهادات (حسب الأكاديمية التابعة له)
  - ملف/بروفايل تعريفي يمكن ظهوره في دليل المعلمين (حسب تفعيل/موافقة)

5) مشرف أكاديمية (Academy Supervisor / Tenant Manager):

- مسؤول داخل أكاديمية محددة فقط:
  - يضيف/يحذف/يدير معلمين داخل الأكاديمية
  - يخصص لهم الخدمات ضمن الحدود المسموحة من المدير
- لا يستطيع التحكم في ملكية النظام أو إعدادات الأكاديميات الأخرى أو أي إعداد عام.

ملاحظات أمنية:

- مطلوب Audit Log يسجل أي عمليات حساسة:
  - إضافة/حذف مستخدم
  - تغيير صلاحيات
  - تفعيل/تعطيل خدمات
  - اعتماد/رفض بروفايل أو محتوى
  - خاصية “الدخول كمعلم” (Impersonation) للمدير للدعم (مذكورة أدناه)

=====================================================
3) إنشاء حساب المعلم + خاصية Access كالإدارة (مثل praply)

المطلوب دعم أكثر من طريقة لإنشاء حساب المعلم:
A) المدير/المشرف ينشئ الحساب للمعلم:

- إرسال دعوة للمعلّم لإكمال البيانات

B) المعلم ينشئ حسابه بنفسه (Self Signup):

- يدخل حالة Pending Approval
- المدير/المشرف يعتمد الحساب → بعدها يظهر في المنصة/الدليل حسب الإعدادات

C) مشرف الأكاديمية يضيف معلم لأكاديميته:

- ضمن نطاق أكاديميته فقط

خاصية “Access/Impersonation”:

- المدير يستطيع الضغط على “Access” للدخول إلى حساب المعلم لأغراض الدعم والمتابعة
- يفضّل أن تكون Read-only افتراضيًا أو على الأقل يتم تسجيل كل شيء في Audit Log:
  - من بدأ access؟ لمين؟ وقت البداية والنهاية؟ وأي تغييرات تمت

=====================================================
4) الخدمات الرئيسية (Modules)

(أ) خدمة المناهج التفاعلية (Curriculum - محور رئيسي)

- المناهج مقسمة:
  - مستويات (Levels)
  - وحدات/أبواب (Units)
  - دروس/إرشادات/مهام (Lessons/Guides/Tasks)
- تتبع التقدم (Progress Tracking) لكل معلم
- إمكانية تخصيص/توجيهات للمناهج داخل الأكاديمية
- تقارير تقدم مفصلة للمدير/المشرف/مشرف الأكاديمية

(ب) خدمة كورسات مسجلة (Recorded Courses)

- كورسات فيديو مسجّلة داخل وحدات
- بعد كل فيديو:
  - اختبار/Quiz (إجباري)
  - تقييم/Score + نجاح/رسوب
- النظام يسجل:
  - من شاهد الفيديو؟ نسبة المشاهدة/الإكمال (إذا أمكن من مشغل الفيديو)
  - نتائج الاختبارات
- تقارير أسبوعية تفصيلية للمدير/المشرف:
  - قائمة المعلمين النشطين والمتأخرين وغير النشطين
  - نسبة الإكمال لكل معلم
  - درجات الاختبارات
  - ملخص تقدّم على مستوى الأكاديمية

(ج) خدمة الشهادات (Certificates) متعددة الأكاديميات

- لكل أكاديمية قوالب شهادات منفصلة تمامًا:
  - مثال: أكاديمية “علي” لها قالب باسم وتوقيع علي
  - أكاديمية “أحمد” لا ترى قالب علي إطلاقًا
- لوحة لإدارة القوالب لكل أكاديمية:
  - رفع قالب / إنشاء قالب
  - متغيرات: اسم المعلم، اسم الكورس، التاريخ، رقم شهادة
- توليد شهادة عند استيفاء شروط (إكمال منهج/نجاح اختبارات…)
- إخراج الشهادة PDF + سجل الشهادات

(د) نظام اجتماعات مباشر (بديل Zoom) عبر API/SDK
لا نريد Zoom لأنه مكلف، نريد منصة مشابهة عبر API/SDK وتكون مدمجة في موقعنا:
المطلوب:

- Meeting فوري أو مجدول
- سبورة تفاعلية (Interactive Whiteboard)
- تقرير عن الحصة:
  - حضور (Attendance) + مدة الحصة + المشاركين
- ملخص للحصة + تفريغ/نص الكلام (Transcript) إن أمكن
- ربط الجلسة بمعلم/أكاديمية + صفحة تفاصيل الحصة داخل النظام

ملاحظة للشركة:

- قد يحتاج الأمر لمزود Video API + مزود Whiteboard SDK (إذا لم يكن المزود الواحد يقدم الاثنين)
- الملخص/الـ transcript يمكن تنفيذها كمرحلة ثانية حسب التكلفة

(هـ) دليل المعلمين (Teacher Directory / Profiles) + فلترة + تواصل
هذه خدمة إضافية مطلوبة:

- المعلم يكون له بروفايل:
  - صورة + فيديو تعريفي
  - تخصصات + خبرة + مواد
  - توفر/Availability (اختياري)
- أصحاب الأكاديميات يستطيعون البحث عن معلمين عبر صفحة دليل معلمين مع فلاتر تساعد الوصول:
  - التخصص/المادة
  - المرحلة
  - الخبرة
  - اللغة
  - لديه فيديو تعريفي (نعم/لا)
  - البلد/المنطقة (اختياري)
  - فرز: الأحدث/الأكثر تطابقًا/الأكثر تفاعلاً
- التواصل:
  - نموذج “طلب تواصل/مقابلة” (Request) من صاحب الأكاديمية إلى المعلم
  - حالات الطلب: New → Accepted → Scheduled → Closed/Hired أو Rejected
  - (اختياري) فتح محادثة داخلية بعد قبول الطلب لمنع السبام
- الخصوصية:
  - Toggle للمعلم: “إظهار في الدليل العام” On/Off
  - السماح بالتواصل من أكاديميات موثقة فقط (اختياري)

=====================================================
5) نظام الخدمات التي تظهر لكل معلم (Entitlements)

نريد أن يكون لكل معلم خدمات مفعلة حسب اشتراكه/قرار المشرف:

- مثال: خدماتنا 3 (منهج تفاعلي + كورسات + اجتماعات) والمعلم مشترك في 2 → يظهر له فقط الاتنين.
  هذا يجب أن يطبق في:
- الواجهة (UI)
- الـ API
- الصلاحيات (يمنع الوصول المباشر للروابط/المسارات غير المفعلة)

=====================================================
6) نظام المدفوعات وتسجيل العمليات تلقائيًا

- أي عملية دفع تتم عبر الموقع يتم تسجيلها تلقائيًا:
  - Transaction ID
  - المستخدم
  - الأكاديمية
  - المبلغ
  - وسيلة الدفع
  - الحالة: Paid/Failed/Refunded
  - تاريخ ووقت العملية
- استخدام Webhooks من بوابة الدفع لتحديث حالة الدفع داخل النظام
- لوحة للمحاسب للمتابعة والتقارير والتصدير

=====================================================
7) التقارير والتحليلات المطلوبة (Reports & Analytics)

تقارير أسبوعية وإدارية (للمدير/المشرف/مشرف الأكاديمية):

- مناهج:
  - نسبة تقدم كل معلم
  - من توقف/تأخر
- كورسات مسجلة:
  - من شاهد الفيديوهات ومن لم يشاهد
  - درجات الاختبارات لكل فيديو/وحدة
  - مؤشرات أسبوعية لمتابعة التطور
- جلسات مباشرة:
  - تقارير حضور
  - مدة الحصص
  - ملخصات الحصص (إذا نفذت)
- دليل المعلمين:
  - عدد بروفايلات المنشورة
  - عدد طلبات التواصل/المقابلات

=====================================================
8) المتطلبات التقنية والتشغيلية (Non-Functional)

- Multi-tenant: عزل بيانات كل أكاديمية
- RBAC + Entitlements + Audit Logs
- Mobile-friendly (خصوصًا لأصحاب الأكاديميات والمعلمين)
- نظام إشعارات داخلية + Email (اختياري)
- رفع ملفات آمن (صور/فيديو/قوالب شهادات)
- إمكانية تصدير التقارير Excel/PDF
- بنية قابلة للتوسع (خاصة الفيديو والبحث بالفلاتر)

=====================================================
9) مراحل التنفيذ المقترحة (لتسعير أوضح)

Phase 1 (MVP):

- Multi-tenant + Roles (مدير/محاسب/مشرف/مشرف أكاديمية/معلم)
- إدارة المستخدمين + تفعيل الخدمات للمعلمين
- المناهج التفاعلية + تتبع التقدم
- كورسات مسجّلة + اختبارات بعد كل فيديو + تقارير أسبوعية
- الشهادات بقوالب منفصلة لكل أكاديمية + PDF
- نظام مدفوعات + تسجيل تلقائي + لوحة المحاسب
- دليل المعلمين (بروفايل + صورة/فيديو + فلاتر + طلب تواصل) + Approval
- Admin Impersonation + Audit Log

Phase 2:

- منصة الاجتماعات المباشرة API/SDK + سبورة + تقارير + transcript/summary
- نظام دعم فني/تذاكر إن رغبت
- تحسينات بحث (Elastic/Meilisearch) إذا زاد الحجم

=====================================================
10) نقاط نحتاج قرار/اقتراح من المطور

- اقتراح مزود Video Meeting API/SDK بديل Zoom (تكلفة أقل) + دعم السبورة
- اقتراح مزود تخزين/بث للفيديوهات (Streaming + tracking)
- اقتراح بوابة الدفع المناسبة
- تقدير زمن وتكلفة Phase 1 وPhase 2
- توضيح خطة الأمان والصلاحيات والتسجيل (Audit)

هذه هي المتطلبات الحالية. المطلوب منك:

1) عمل تصور UI/UX مبدئي (Wireframes) لكل Dashboard والصفحات الأساسية
2) عرض فني وتقني بالتفاصيل + التكلفة + الجدول الزمني
3) تحديد ما يتم في Phase 1 وما يتم في Phase 2 بوضوح
